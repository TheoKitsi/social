# CMP-2.3.1: Particularities Catalog

## Functions

| FN-ID | Function |
|---|---|
| **FN-2.3.1.1** | The system shall provide predefined particularity categories: lifestyle particularities (e.g., crossdressing, conviction-based veganism, minimalism), sexual preferences (e.g., BDSM, asexuality), unconventional life models (e.g., past open relationship, tiny house living), special family bonds (e.g., close grandparent bond, caregiving responsibility). |
| **FN-2.3.1.2** | The system shall provide a free-text field: "What most people don't know about me but my partner must know." |
| **FN-2.3.1.3** | The system shall integrate particularities into the matching algorithm as a compatibility factor — both positive match (both share the trait) and acceptance (partner has no rejection). |
| **FN-2.3.1.4** | The system shall set particularity fields to visibility level 1 by default (algorithm only, never visible in profile views), with the option for the user to consciously increase the level. |
